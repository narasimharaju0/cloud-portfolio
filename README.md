# ☁️ Narasimha Raju | Multicloud & DevOps Engineering Portfolio

A premium, production-grade Single-Page Application (SPA) designed to showcase cloud infrastructure architecture, automated deployment pipelines, and core DevOps engineering capabilities. 

Built with a terminal-inspired dark grid aesthetic, this portfolio operates as a highly optimized, interactive resume tailored for engineering leaders and technical recruiters.

🔗 **[View Live Deployment](https://cloud-portfolio-theta.vercel.app)**

---

## 🚀 Core Architecture & Features

- 🖥️ **Interactive Linux-Style Terminal:** A custom CLI interface built into the DOM. Users can execute commands (`help`, `resume`, `skills`, `whoami`) to explore technical credentials in a native developer environment.
- 🌐 **Dynamic Infrastructure Topology:** Smooth, interactive node maps powered by Framer Motion, visualizing complex multicloud architectures (AWS, Azure) and microservice routing.
- ⚡ **High-Performance Build:** Bundled with Vite and strictly typed with TypeScript, ensuring zero-layout-shift and sub-second compilation.
- 🔒 **Secure Transmission Channel:** Fully functional, serverless contact gateway utilizing the Formspree REST API for direct, background data transmission.

## 💻 Local Development Setup

To clone and spin up this infrastructure locally, ensure you have Node.js installed, then execute the following commands in your terminal:

```bash
# 1. Clone the repository
git clone [https://github.com/narasimharaju0/cloud-portfolio.git](https://github.com/narasimharaju0/cloud-portfolio.git)

# 2. Navigate into the project directory
cd cloud-portfolio

# 3. Install all required dependencies
npm install

# 4. Boot up the local Vite development server
npm run dev
```
The application will spin up instantly on your local port (typically http://localhost:5173).

## 📂 Repository Structure

**/src/components/** — Modular, reusable React components (Hero, TopologyMap, GlassCard).

**/src/components/sections/** — Main view blocks, including the ResumeTerminal and Contact gateway.

**/src/data/** — Centralized state and configuration files mapping out skills, constants, and the mock-CLI responses.

**/public/** — Static assets, including the downloadable PDF resume and vector graphics.
